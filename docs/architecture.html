Executive Summary

mcp-server-devops is a Model Context Protocol server that exposes six DevOps tools — Docker container management, Docker image building, Helm release listing, Ansible playbook execution, Jenkins build triggering, and Terraform plan summarization — to AI assistants via stdio transport. Built on the MCP SDK with TypeScript, it acts as a bridge between language models and production DevOps tooling, enabling conversational infrastructure operations. Each tool wraps a CLI or API call (Dockerode for container operations, child_process for Helm/Ansible/Terraform, node-fetch for Jenkins REST API) and returns structured JSON results.

Data / Request Flow

  1. AI assistant sends MCP tool-call request over stdin
  2. StdioServerTransport deserializes the JSON-RPC message
  3. McpServer routes to the matching registered tool handler
  4. Tool handler validates input via Zod schema
  5. Handler invokes external system: Dockerode API, child_process exec (helm/ansible/terraform CLI), or node-fetch (Jenkins REST)
  6. Response is structured as JSON and returned as MCP text content
  7. On error, handler returns isError: true with descriptive message

Security Controls

  • Input validation: All tool parameters validated via Zod schemas with typed constraints
  • Jenkins auth: Basic authentication with base64-encoded username:token via Authorization header
  • Filesystem checks: docker_build and ansible_run_playbook verify file existence before execution
  • Buffer limits: child_process executions capped at 10-20 MB maxBuffer to prevent memory exhaustion
  • Timeouts: Ansible playbooks capped at 600s, Terraform plan at 300s
  • Docker socket: Defaults to /var/run/docker.sock; remote hosts connect on port 2375
  • Container isolation: Dockerfile runs production build with --omit=dev dependencies
  • No secrets in code: Jenkins credentials passed as tool parameters, not hardcoded

Production Readiness Checklist

  • Zod schema validation on all tool inputs
  • Structured error handling with isError flag on every tool
  • Multi-stage Docker build with minimal runtime image (node:20-alpine)
  • Production NODE_ENV=production in container
  • Buffer and timeout limits on all child_process calls
  • TypeScript strict mode with declaration maps and source maps
  • devDependencies excluded from production container (--omit=dev)
  • Jenkins auth via per-request credentials (no stored secrets)

Deployment

  • Docker: docker build -t mcp-server-devops . — multi-stage build installs docker-cli, helm, ansible, terraform into alpine runtime
  • Standalone: npm ci && npm run build && node dist/index.js
  • Dev mode: npm run dev (ts-node)
  • MCP client config: Add stdio transport pointing to node dist/index.js or the Docker container
  • Prerequisites: Docker socket access, helm/ansible/terraform CLIs on PATH (or use Docker image)
